# Data Pipeline

The public repository contains processed and anonymized reporting data only. Raw source files are intentionally excluded.

## Flow

```text
Processed/anonymized input structure
        -> src/etl.py
        -> data/processed/clean_data.csv
        -> data/processed/task2_asset_summary.csv
        -> Power Query ingestion
        -> Power BI semantic model
        -> DAX measures
        -> Selected report pages
```

## Python ETL

`src/etl.py` prepares two reporting tables:

- `clean_data.csv`: monthly asset-level reporting table
- `task2_asset_summary.csv`: asset-level summary for follow-up and prioritization

The ETL standardizes market fields, ownership information, production certainty, availability units, budget and actual production, wind impact, residual deviation, and exclusion logic.

## Power Query

Power Query is used for ingestion and schema typing inside the Power BI model. The heavier business-rule transformation is handled in Python so the logic remains repeatable and easier to inspect.

When opening the PBIP project on a new machine, update the `File.Contents(...)` paths in Power Query to point to:

```text
data/processed/clean_data.csv
data/processed/task2_asset_summary.csv
```

## Power BI Model

The current Power BI model includes:

- `clean_data`
- `task2_asset_summary`
- `Variance Step`
- Date table generated by Power BI

The public v0 documents the model structure and DAX measures rather than including the PBIX/PBIP file. The model is intentionally compact and designed to demonstrate reporting logic, KPI definitions, and selected management-facing report pages rather than a full production data platform.
